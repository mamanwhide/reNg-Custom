    Ollama Configuration (Offline LLM)

    Ollama is a local LLM service for offline AI. Models need to be pulled (downloaded) first. Note: Ollama container requires internet connectivity to download models. If pull fails, ensure your Docker environment has network access.

    When enabled: Uses Ollama (if models available), fallback to OpenAI if configured.
    When disabled: Uses OpenAI if key is available, fallback to Ollama if no models needed.
    Recommended: mistral (fast, balanced) or neural-chat (good quality)
    Current setting: {{ ollama_settings.selected_model }}
    Note: Models are automatically pulled when saved (may take several minutes depending on size)
    Recommended Setup & Troubleshooting
    Best Practice:
    • ✅ Configure OpenAI API Key for reliable vulnerability report generation
    • ✅ Set Use Ollama disabled to avoid network issues
    • ℹ️ Use Ollama only if you have offline requirement or network access to pull models
    If Ollama Model Pull Fails:
    • Check Docker network connectivity: docker logs ollama | grep Error
    • Verify Ollama container has internet: docker exec ollama ping google.com
    • Pull models manually: docker exec ollama ollama pull mistral
