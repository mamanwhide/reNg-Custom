    URL Scope and Exclusions

    Starting Point Path (Optional)

    {% if starting_point_path %} {% else %} {% endif %} Defines where the scan should begin. Leave blank to scan from the root (/) and include all subdomains.
    If a path is provided (e.g., /home), the scan will focus only on that path and its subpaths, skipping subdomain scanning. For example, entering '/home' for {{domain.name}} will scan https://{{domain.name}}/home, but not other parts of {{domain.name}} or its subdomains.

    Excluded Paths (Optional)

    Enter paths or regex patterns to exclude from the scan. Type a path or pattern and press Enter to add it. Supports both exact path matching and regex patterns. Examples:
    • /admin excludes paths starting with '/admin'
    • /images/.*\.jpg excludes all .jpg files in the images directory
    • /static/(?:css|js)/ excludes all contents of /static/css/ and /static/js/
    Common exclusions:
    • Static assets: /images/.*, /css/.*

    Note: Use regex patterns carefully. While exclusions can speed up scans, be cautious not to exclude critical areas that may contain vulnerabilities. Test your patterns to ensure they match as intended.

    Proxy Setup

    Choose whether scan traffic should be routed through a proxy. Proxies help bypass IP-based rate limits and geo-restrictions, but they route all traffic through a third-party server and may slow down local (LAN) scans significantly.

    Scan traffic is routed through a randomly selected proxy from your proxy list. Best for scanning external targets to avoid rate-limiting or IP blocks.

    Scan traffic goes directly to the target. Recommended for LAN/intranet targets or when your network connection is already fast and reliable.

    Using Proxy — Known risks:
    • LAN / intranet scans will be unusably slow — all packets leave your machine, go through the proxy server, and return, adding 100–500 ms RTT per request.
    • Privacy: proxy operators can observe the URLs and payloads being scanned. Avoid using free public proxies against sensitive internal systems.
    • Reliability: free proxy lists contain dead entries; paraKang auto-selects a live one, but some tools may still fail if the proxy drops mid-scan.
    • Throughput: bandwidth is limited by the proxy’s uplink. Large scans (screenshots, directory fuzzing) take longer.
    No Proxy — Things to keep in mind:
    • Your real IP is exposed to the target. Public targets may rate-limit or block your IP after repeated scans.
    • Google Dorking (OSINT) will be performed from your IP — Google may temporarily block it after a few searches.
    • Best choice for LAN / intranet targets where proxies add unnecessary latency.
